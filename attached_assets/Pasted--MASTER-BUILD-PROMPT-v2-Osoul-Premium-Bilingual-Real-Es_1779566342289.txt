## 4. DESIGN SYSTEM

### 4.1 Color Palette — "Navy, Gold, Ivory"
A premium, capital-investor palette. Deep navy carries authority; antique gold (not bright/yellow gold) carries heritage; warm ivory keeps it from feeling cold or corporate.

```css
/* Primary palette */
--color-navy:        #0B1E3F;   /* Brand primary — deep, established */
--color-navy-2:      #14305C;   /* Hover, elevated surfaces */
--color-navy-3:      #1B3E73;   /* Tertiary backgrounds */
--color-gold:        #B8893E;   /* Primary accent — antique, never bright */
--color-gold-2:      #D4A95C;   /* Hover state for gold */
--color-gold-soft:   #E8D8B0;   /* Background tints */
--color-ivory:       #F8F4EA;   /* Page background */
--color-cream:       #FDFBF5;   /* Card backgrounds */

/* Text */
--color-ink:         #0A0F1F;   /* Primary text — near-black navy */
--color-ink-2:       #4A5468;   /* Secondary text */
--color-ink-3:       #8892A6;   /* Muted text, captions */
--color-line:        #E5DDC9;   /* Dividers, hairlines */

/* Operation tags (the colored chips on cards) */
--tag-sale:          #2E5C3E;   /* "للبيع" — confident green */
--tag-rent:          #8B6F47;   /* "للإيجار" — warm bronze */
--tag-wanted:        #6B4226;   /* "مطلوب للشراء" — deep terracotta */
--tag-exchange:      #4A5C7C;   /* "بدل" — slate */
--tag-partner:       #7C5C4A;   /* "مشاركة" — taupe */
```

### 4.2 Typography

**Arabic stack:**
- Display (hero, section headings): **"Cairo"** weights 800–900 — bold, confident, generous
- Body: **"Tajawal"** weights 400–600 — clean, modern, highly readable
- Numerals (Arabic-Indic) inline with body: same as body

**English / Latin stack:**
- Display: **"Cormorant Garamond"** weights 500–700 — serif, premium, balances Arabic Cairo
- Body: **"Inter"** weights 400–600 — neutral, professional
- Tracked caps (small labels, "EST. 2010"): Inter weight 600, letter-spacing 0.15em

**Rules:**
- Arabic body line-height: **1.85** (Arabic needs more leading than Latin)
- English body line-height: 1.6
- Headings have tight tracking: Arabic letter-spacing 0, English letter-spacing -0.01em
- The phone number `+201020210991` ALWAYS renders in Inter, LTR, even inside Arabic flow — wrap in `<bdi>` and `direction: ltr`

### 4.3 Spacing & Layout
- 4px grid. Section padding: 120px desktop / 64px mobile.
- Max content width: **1280px**, centered.
- Generous whitespace — premium brands breathe.
- Border radius: **8px** on cards, **6px** on buttons, **999px** on chips/tags. (Less rounded than v1 — navy/gold reads more refined with subtler curves.)
- Shadows: refined and subtle.
  - Card resting: `0 1px 3px rgba(11,30,63,0.06), 0 1px 2px rgba(11,30,63,0.04)`
  - Card hover: `0 12px 32px rgba(11,30,63,0.10), 0 4px 8px rgba(11,30,63,0.06)`
- **Gold hairlines:** use 1px gold lines (`--color-gold` at 40% opacity) as subtle dividers between sections — premium signature.

### 4.4 Texture & Decorative Elements
- Section dividers occasionally use a **thin gold hairline (1px) flanked by two small gold diamonds (◆)** — classic Arabic editorial design touch.
- Hero and About page have a **subtle pattern overlay** (3% opacity) — an Islamic geometric tessellation (8-point star pattern) in gold over navy. Generate as SVG. Use sparingly — accent, not background.

### 4.5 Direction & Internationalization
- Arabic version: `<html lang="ar" dir="rtl">`
- English version: `<html lang="en" dir="ltr">`
- Mirror EVERY layout element: nav order, breadcrumbs, card internals, icon positions.
- Use Tailwind logical properties (`ps-`, `pe-`, `ms-`, `me-`, `start-`, `end-`) — never `pl-`, `pr-`, `ml-`, `mr-`.

## 5. TECH STACK & ARCHITECTURE

- **Framework:** Next.js 14 (App Router) + TypeScript
- **Internationalization:** **`next-intl`** with locale segments: `/ar/...` and `/en/...`. Default redirect from `/` → `/ar`.
- **Styling:** Tailwind CSS + `tailwindcss-rtl` plugin
- **Animations:** Framer Motion (entrance, scroll-triggered, hover)
- **Icons:** Lucide React
- **CSV parsing:** PapaParse — runs at build time, outputs a typed `data/properties.ts` file. **No runtime CSV fetching.**
- **Fonts:** Google Fonts CDN (Cairo, Tajawal, Cormorant Garamond, Inter) with `font-display: swap`
- **i18n message files:** `messages/ar.json` and `messages/en.json` — every string on the site lives here
- **Deployment target:** Vercel (static-export ready)
- **Performance budget:** Lighthouse ≥95 on all four metrics

**Instructions to the AI agent:**

Generate 52 unique testimonials, originally written in Egyptian colloquial Arabic. The goal is **authenticity**, not praise saturation.

## 13. DATA HANDLING — CSV PROCESSING

When parsing the attached `عروض_العقارات.csv`:

1. **Strip the UTF-8 BOM** (`﻿`) at file start
2. Extract these 5 columns per row: `التصنيف`, `النوع`, `العملية`, `الموقع`, `المساحة`
3. **Do not import** `السعر`, `التفاصيل`, `رقم التواصل` — they must not exist in the typed object
4. Replace `"غير محدد"` in `الموقع` with display value: "موقع عند الاستفسار" / "Location on inquiry"
5. Replace `"غير محدد"` in `المساحة` with display value: "حسب الاستفسار" / "On inquiry"
6. **Split compound operations** like `"للبيع / مشاركة"` on `/` — row matches both filters
7. **Split compound classifications** like `"زراعي / صناعي"` on `/` — row belongs to both categories AND to the "مختلط / Mixed" super-category
8. Generate a **stable offer number** (1-based index, zero-padded, formatted `#٠٠١` in AR / `#001` in EN) for each row
9. **Normalize area strings** for the slider:
   - `"650 متر"` → `{value: 650, unit: "m"}`
   - `"20 فدان"` → `{value: 20, unit: "feddan"}`
   - `"440 متر (200 مباني)"` → `{value: 440, unit: "m"}` (use the larger/total number)
   - `"6 فدان و 3 قيراط"` → `{value: 6.125, unit: "feddan"}` (24 qirat = 1 feddan)
   - `"2 فدان وعنبر 15 ألف طائر"` → `{value: 2, unit: "feddan"}` (ignore the secondary count)
10. **English translations**: build a translation map for:
    - Categories: سكني → Residential, زراعي → Agricultural, صناعي → Industrial, مختلط → Mixed
    - Operations: للبيع → For Sale, للإيجار → For Rent, مطلوب للشراء → Wanted, بدل → Exchange, مشاركة → Partnership
    - Common types: فيلا → Villa, بيت → House, شقة → Apartment, عمارة → Building, قطعة أرض → Plot, مزرعة → Farm, أرض زراعية → Agricultural Land, مصنع بلاستيك → Plastic Factory, مصنع كيماوي → Chemical Factory, مصنع غذائي → Food Factory, مصنع أسمدة كيماوي → Chemical Fertilizer Factory, عنبر دواجن → Poultry Barn, أرض مصنع → Factory Plot, مزرعة دواجن → Poultry Farm, مشروع متكامل → Integrated Project, شقق مفروشة → Furnished Apartments
    - Locations: keep Arabic zone numbers as "Zone N" in English (المنطقة ١٣ → Zone 13), translate place names (الخطاطبة → Khatatba, وادي النطرون → Wadi El Natrun, الحزام الأخضر → El-Hizam El-Akhdar / Green Belt, بيت الوطن → Beit El Watan, النخيل → El-Nakhil, الروضة → El-Rawda, المربع الذهبي → Golden Square, المطورين → El-Motawereen, العامرية → El-Ameriya)
11. Output → typed `data/properties.ts`. Re-running the build with an updated CSV regenerates everything.

### 15.3 Locale-aware formatting
- **Numbers:** Arabic-Indic digits (٠١٢٣٤٥٦٧٨٩) in Arabic body text; Latin digits (0123456789) in English

- **Dates:** Arabic months (يناير، فبراير...) in AR; English months (January, February...) in EN
- **Direction:** `<html dir="rtl">` in AR, `<html dir="ltr">` in EN; Tailwind logical properties everywhere

### 15.4 Translation completeness
EVERY string on the site must exist in both `ar.json` and `en.json`. Including:
- Nav items, button labels, filter labels, empty states, error states
- Category names, operation names, property types, location names (see §13 step 10)
- Footer text, hours, address label
- The signature promise (in both languages)
- The 52 reviews remain in original Arabic, with small English summary translations beneath them on the EN page only
